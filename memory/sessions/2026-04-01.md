## 2026-04-01 23:15 MSK — Logout made deterministic with a server auth endpoint

### Context

- The user reported an intermittent bug after logout: sometimes the app landed on `/` and showed the post-auth action screen instead of going to `/login`.
- Middleware protects both `/` and `/login`, so this symptom pointed to a race where logout was not fully reflected in the auth cookies by the time the redirect happened.

### What Changed

- Diagnosed the room logout flow and confirmed it used browser-only `supabase.auth.signOut()` followed by `router.push('/login')`.
- Confirmed middleware behavior:
  - `/login` redirects to `/` when a user is still present
  - `/` redirects to `/login` only when middleware sees no user
- Added a dedicated server route:
  - `app/api/auth/logout/route.ts`
- Updated room logout to:
  - call `POST /api/auth/logout`
  - clear local saved rooms
  - `router.replace('/login')`
- This makes logout clear Supabase auth cookies on the server before the client navigates.

### Relevant Paths

- `app/api/auth/logout/route.ts`
- `app/room/[code]/page.tsx`
- `middleware.ts`

### Verification

- `npx tsc --noEmit`
- `rg -n "auth\\.signOut\\(|/api/auth/logout" app components lib`

### Result

- Logout no longer depends on a client-only session update.
- Middleware should now consistently treat the user as logged out before the redirect to `/login`.

## 2026-04-01 23:40 MSK — Logout hardened against cached App Router payloads

### Context

- After the server logout endpoint was added, the user still sometimes landed on `/` and saw the post-auth action screen.
- The screenshot showed `401` on `/api/rooms` while `/` was already rendered, which means the session was gone but the client still displayed a cached authenticated root route.

### What Changed

- Diagnosed this as an App Router cache/navigation issue rather than a remaining auth-cookie problem.
- Updated room logout to:
  - set `leavingRef.current = true`
  - call `POST /api/auth/logout`
  - clear local saved rooms
  - perform `window.location.replace('/login')`
- This bypasses client-side route cache reuse and forces a fresh server-guarded navigation.

### Relevant Paths

- `app/room/[code]/page.tsx`
- `app/api/auth/logout/route.ts`

### Verification

- `npx tsc --noEmit`

### Result

- Logout now uses a hard redirect after the server sign-out completes.
- This prevents stale cached `/` payloads from rendering after the session has already been cleared.
